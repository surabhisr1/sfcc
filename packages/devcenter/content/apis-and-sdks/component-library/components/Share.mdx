### Basic example

```jsx react-live=true
class StatefulExample extends React.Component {
    constructor() {
        this.state = {
            open: false
        }
    }

    render() {
        return (
            <Share
                open={this.state.open}
                onShow={() => this.setState({open: true})}
                onDismiss={() => this.setState({open: false})}
            />
        )
    }
}
```

### With triggerElement

The `triggerElement` prop allows for an existing or custom component to trigger the `Share` sheet.

_With a custom component_

```jsx react-live=true
class StatefulExample extends React.Component {
    constructor() {
        this.state = {
            open: false
        }
    }

    render() {
        return (
            <Share
                open={this.state.open}
                onShow={() => this.setState({open: true})}
                onDismiss={() => this.setState({open: false})}
                triggerElement={
                    <button>
                        <IconLabel label="Share" iconName="share" />
                    </button>
                }
            />
        )
    }
}
```

### Potential uses

-   The UI associated with the share interaction is most commonly found on the product details page.
-   Typically used after the share interaction, from within the overflow menu. For apps launched from the homescreen, this is usually found in the header bar.

### Best practices

-   Any prompt to share should be presented inline, next to the content that a user may want to share. For products, sharing should be presented as a secondary action, prioritizing the UI for adding the item to the user's cart or wishlist.
-   Use [Google's Material Design Icons](https://material.io/icons/) for the actions within the share dialogue to match the UI of the device.
