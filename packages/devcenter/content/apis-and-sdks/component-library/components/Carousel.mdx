<div class="component-intro">

A carousel is used to display a series of content in a slideshow format. It shows one item at a time and allows users to cycle through the items by swiping, or using their next/previous button.

Image Carousels allow the display of multiple images in a constrained space, typically with controls to allow the user to flip between them.

The `Carousel` component is a wrapper that contains [CarouselItem](CarouselItem) components.

Related component: [CarouselItem](CarouselItem).

            'http://res.cloudinary.com/monsterpetsupplies/image/upload/' +
            'c_pad,w_500,h_500/v1456507382/Product/Boris_Side_On_Eating_Sealed_' +

    render() {
        return (

### Potential uses

-   On a product details page, to enable the display of multiple images.
-   On a homepage, to rotate through promotional graphics.

### Accessibility

-   Carousels typically contain images controlled on the client side. In order to ensure an accessible color contrast, navigation controls should be positioned on top of a suitable background color.
-   If you're using Pips, ensure the active and non-active states are differentiated by more than opacity or color. For example, use a different border, or a variation in size.

### Best practices

-   Tapping on-screen controls can be used to scroll forward and backward through the selection of images.
-   Swiping left and right can also be used to scroll forward and backward through the selection of images.
-   Using swiping to advance images is useful and expected behavior, but it's still good practice to provide visual indicators of user actions, such as previous/next arrow indicators.
-   It's best to indicate the number of elements in a carousel. You can do that using Pips (multiple dots positioned below the carousel), a thumbnail gallery, or a text indicator (such as "1 of 4").
-   Consider using a carousel for a single image if you expect other images may be added in the future.
